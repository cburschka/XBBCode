1 # Extensible BBCode
2 
3 This is a BBCode parser for Drupal that can be extended with custom tag macros.
4 If you install it on your Drupal site, it will create a text format named
5 "BBCode" that generates HTML out of text markup such as this:
6 
7  This is [b]bold[/b] and [url=http://drupal.org/]this is a link[/url].
8 
9 Custom tags use the [Twig](http://twig.sensiolabs.org/) template engine included
10 in Drupal's core.
11 
12 # Developing
13 
14 Aside from creating custom template-based tags, you can also provide tags from a
15 module. There are two ways to do so:
16 
17 1. Define a template tag using a configuration file.
18 2. Implement a full-featured tag plugin class.
19 
20 ## Template
21 
22 For most use cases, a template is sufficient. Twig templates support simple
23 control structures that mean the tag output can be dynamic without needing PHP.
24 
25 A template tag is defined in a file named `config/install/xbbcode.tag.{id}.yml`
26 that must contain the following:
27 
28  id: {id}
29  label: "An administrative label for your tag."
30  description: "Describes the tag's function for users (used for filter tips)."
31 
32  # A default name for the tag (as in [name]...[/name]).
33  name: {name}
34 
35  # A sample use case for the tag (for the filter tips).
36  sample: "[{{ name }}]...[/{{ name }}]"
37 
38 It must also contain exactly one of the following:
39 
40  # An inline Twig template, equivalent to a tag created through the site:
41  template_code: "<span>{{ tag.content }}</span>"
42 
43 OR
44 
45  # A template file that must be placed in "templates/xbbcode-tag-{id}.html.twig"
46  template_file: xbbcode-tag-{id}.html.twig
47 
48 In the latter case, the template must also be registered with the theme registry
49 by implementing `hook_theme()`. Since the template is only used directly, only
50 the theme hook is required:
51 
52  function {module}_theme() {
53  return [
54  'xbbcode_tag_{id}' => [],
55  ];
56  }
57 
58 If your tag is self-closing or "empty" (in that it consists only of an opening
59 tag like `[hr]`), it must contain the following:
60 
61  selfclosing: true
62 
63 Optionally, you may declare attachments (such as
64 [CSS/JS libraries](https://www.drupal.org/developing/api/8/assets) defined in
65 `*.libraries.yml`) that will be added whenever the tag is rendered:
66 
67  attached:
68  # see
69  library:
70  - module/library

72 ## Plugin class
73 
74 A plugin class can use PHP while processing a tag, and is therefore more
75 powerful.
76 
77 BBCode tags are
78 [Annotations-based plugins](https://www.drupal.org/node/1882526). To provide
79 one, your module needs to contain a class like this (in the appropriate PSR-4
80 path `src/Plugin/XBBCode/`).
81 
82  namespace Drupal\{module}\Plugin\XBBCode;
83 
84  use Drupal\xbbcode\Plugin\TagPlugin;
85  use Drupal\xbbcode\ElementInterface;
86 
87  /**
88  * @XBBCodeTag(
89  * id = "xbbcode.url",
90  * title = @Translation("Link"),
91  * description = @Translation("This creates a hyperlink."),
92  * name = "url",
93  * sample = @Translation("[{{ name }}=http://www.drupal.org/]Drupal[/{{ name }}]")
94  * attached = {
95  * "libraries" = {
96  * "module/library"
97  * }
98  * }
99  * )
100  */
101  class YourTagPlugin extends TagPlugin {
102  /**
103  * {@inheritdoc}
104  */
105  public function process(ElementInterface $tag) {
106  return '<em>' . $tag->content() . '</em>';
107  }
108  }
109 
110 The `{{ name }}` placeholder is required as the tag name is configurable.
111 
112 The required function `TagPluginInterface::process(ElementInterface $tag)`
113 receives a tag occurrence as encountered in text, and must return HTML code.
114 
115 The `ElementInterface` object provides the following methods:
116 
117 - `content()` returns the rendered content of the tag.
118 - `option()` returns the string following "=" in the opening tag, such as
119  [url=URL]...[/url]
120 - `attr($name)` returns the value of a named attribute in the opening tag,
121  such as [quote author=AUTHOR date=DATE]...[/quote]
122 - `source()` returns the unrendered content of the tag. This can be used when
123  your tag's content should not be rendered, such as [code]...[/code].
124 - `outerSource()` returns the full source, including opening and closing tags.
125  This can be returned if you want to leave the tag unrendered.
